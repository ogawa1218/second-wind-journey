# garmin-post スキル

Garmin のスクリーンショット画像からThreads投稿文とSubstack週次ログ行を自動生成する。

## 発動条件

ユーザーが `/garmin-post` を実行したとき、または Garmin のスクリーンショット画像を貼り付けて投稿文生成を依頼したとき。

---

## サブエガ（2:50:00切り）VO2max 目標値の定義

投稿文の「考察」で必ず参照する基準。VO2max は ml/kg/min（Garmin推定値）。
**体重が落ちると分母(kg)が小さくなりVO2max/kgは自動的に上がる** ため、64kg計画との連動が重要。

| 区分 | VO2max | 想定フルタイム（Garmin予測ベース） | コメント |
|---|---|---|---|
| 現在地 | **59** | 約2:48 | 既にサブエガ射程内。ここを土台に伸ばす |
| 中間チェック | **62** | 約2:42 | 中盤の失速耐性・余裕が増す通過点 |
| **ゴール目標** | **65** | 約2:36 | サブエガを盤石に超える有酸素出力 |

**運用ルール:**
- 投稿の考察では「現在値 → ゴール65までの残り」を毎回明示する。
- VO2max が読み取れた回は、前回比の増減と「64kg到達時の理論値」をセットで触れる。
  - 簡易換算: `理論VO2max@64kg ≒ 現VO2max × (現体重kg ÷ 64)`（最高酸素摂取量[L/min]が一定と仮定した粗い目安。あくまで動機づけ用で断定しない）
- レース日 2026-11-22 まで逆算し、ゴール65に対して「順調／要加速／黄信号」の3段階で評価する（59は既にサブエガ可能圏なので、65は"盤石化"の攻めの目標）。

## 実行手順（必ず順番通りに行う）

### Step 1 — 現在日付を取得する

```bash
date +%Y-%m-%d
```

**絶対に会話の記憶や推測から日付を決めてはいけない。** 必ず上記コマンドの出力を使う。

### Step 2 — Day番号を計算する

- `DAY1 = 2026-06-12`（固定値）
- `TODAY = Step 1 で取得した日付`
- `DAY_NUMBER = (TODAY - DAY1).days + 1`

Bash で計算する場合:
```bash
echo $(( ( $(date -d "TODAY" +%s) - $(date -d "2026-06-12" +%s) ) / 86400 + 1 ))
```

### Step 3 — 画像を読み取る

ユーザーが提供した Garmin スクリーンショットから以下を抽出する。
値が読み取れない場合は `—` とし、推測しない。

| 項目 | 変数名 | 単位・形式 |
|---|---|---|
| 距離 | `DISTANCE` | km（小数点1桁、例: 12.3） |
| 平均ペース | `AVG_PACE` | 分:秒/km（例: 5'30"） |
| 平均心拍数 | `AVG_HR` | bpm（整数） |
| 経過時間 | `DURATION` | h:mm:ss または mm:ss |
| 最高心拍数 | `MAX_HR` | bpm（あれば） |
| 高度上昇 | `ELEVATION` | m（あれば） |
| カロリー | `CALORIES` | kcal（あれば） |
| VO2max | `VO2MAX` | ml/kg/min（整数、Garmin「VO2 Max」/「最大酸素摂取量」表示。あれば） |
| 体重 | `WEIGHT` | kg（小数1桁。Garmin/体組成計の値、なければStep4で確認） |
| 睡眠スコア | `SLEEP_SCORE` | pt（Garmin睡眠タブのスコア） |
| 睡眠時間 | `SLEEP_H` | h（小数1桁） |

### Step 4 — 不足情報をユーザーに確認する

以下の情報が画像から読み取れない、または文脈から明らかでない場合のみ質問する。
**すでにわかっている項目は聞かない。**

```
以下を教えてください（わからない項目は空白でOK）:

1. 種別: [ イージー / 閾値走 / ロング走 / インターバル / MP走 / テンポ走 / レース / その他: ]
2. 今日の体重: ___kg（前日比 ±___kg、スタート時100kgからの累計 -___kg）
3. 睡眠: ___h・睡眠スコア___pt（Garmin「睡眠」タブのスコア、またはBody Battery起床時値）
4. VO2max: ___ml/kg/min（Garmin最新値。前回から変化があれば前回値も）
5. 一言メモ: ___（省略可）
```

### Step 5 — 種別の判定（自動判定ルール）

ユーザーが種別を指定しなかった場合、平均ペースから自動判定する:

| 平均ペース | 自動判定種別 |
|---|---|
| 6:00/km 以上 | イージー |
| 5:00〜5:59/km | イージー〜テンポ境界（確認推奨） |
| 4:20〜4:59/km | テンポ走 / 閾値走 |
| 4:00〜4:19/km | MP走 |
| 3:59/km 以下 | インターバル / レースペース |

距離で補正:
- 25km 以上 → 「ロング走」を優先表示
- 5km 未満 + 速い → 「5km TT」の可能性を提示

## Step 6 — Threads 投稿文を生成する

### テンプレート構造

```
Day{DAY_NUMBER}/164 🏃{DISTANCE}km＋{種別}
{体重情報行}
{睡眠情報行}
{VO2max情報行}
{状況1行}
{考察ブロック（体重・睡眠・VO2maxを1〜3行で）}
→{仕組みがどう機能したか1行}
```

### 各行の生成ルール

**体重情報行:**
- 体重あり: `体重{今日の体重}kg（前日比{±X}kg／スタートから累計{-XX}kg）`
- 体重なし: 行ごと省略

**睡眠情報行:**
- 睡眠あり: `睡眠{X}h・スコア{N}pt`
- 睡眠スコアのみ（時間不明）: `睡眠スコア{N}pt`
- 睡眠情報なし: 行ごと省略

**VO2max情報行:**
- VO2maxあり: `VO2max{N}（ゴール65まであと{65-N}）`
  - N≧65 の場合: `VO2max{N}（ゴール65到達・貯金{N-65}）`
- VO2maxなし: 行ごと省略

**考察ブロック（このスキルの肝・必ず生成する）:**
体重・睡眠スコア・VO2max のうち**読み取れた指標それぞれ**について、データ→意味づけを1行ずつ。
「行動序列ドミノ（睡眠→食事→運動）」の文脈に必ず接続する。断定せず観測ベースで書く。

- **体重の考察:** 64kg計画に対する位置づけ。
  例: `残り{現体重-64}kgを{レースまで週数}週で。週−{必要ペース}kgなら計画通り。`
  例: `体重−1kgでフルは約3分速くなる試算。64kgなら今より{(現体重-64)*3}分の貯金。`
- **睡眠の考察:** スコアと当日の練習強度の整合。
  例: `スコア{N}ptは回復十分→ポイント練GOの根拠。` / `スコア{N}pt低め→ドミノ通り強度を一段落として正解。`
- **VO2maxの考察:** ゴール65への距離＋体重連動。
  例: `現{N}→ゴール65まで{65-N}。64kg到達時の理論値は約{現VO2max×現体重/64}で、減量がそのまま有酸素エンジンの底上げになる。`
  例: `前回{前回値}→今回{N}（{±X}）。{レース日まで日数}日で65なら「{順調/要加速/黄信号}」。`

VO2maxが無い回は体重・睡眠の2軸で考察する。1行も書けるデータが無い回のみ考察ブロックを省略。

**状況1行:**
画像から読み取ったデータを自然な日本語1文にまとめる。
形式: `{距離}km・{ペース}/km・心拍{HR}bpm・{所要時間}`
例: `12.3km・5'30"/km・心拍148bpm・67分完走`

**仕組みがどう機能したか1行:**
以下のif-thenルールのうち、**今回の状況に最も当てはまるもの**を引用して1文にする。
ユーザーのメモがある場合はそれも参考にする。

```
参照するルール候補:
- 「睡眠5時間未満→ポイント練をイージーに自動格下げ」
- 「2日連続ゼロ→3日目は5分着替えて外に出る」
- 「大雨→トレッドミル/階段10分×2+筋トレ」
- 「子供が早朝起床→朝枠消滅を即受容、昼or夜に移動」
- 「ネックチェック：首から上のみ→20分ジョグ可」
- 「飛ばした練習を翌日に上乗せしない（no_debt_rule）」
- 「前夜バナナ+プロテインで質練準備（carb periodization）」
- 「wake_time 04:00 habit が機能した」
- 特に何もなければ: 「if-thenルール通りの朝ルーティンが機能した」
```

### 投稿文の文字数チェック

Threads は500文字上限。生成後に文字数を確認し、超過する場合は状況1行を短縮する。
**ハッシュタグは付けない**（ハンドルに統一）。

### 完成例

```
Day 13/164 🏃12.3km＋イージー
体重68.8kg（前日比-0.2kg／スタートから累計-31.2kg）
睡眠6.5h・スコア74pt
VO2max59（ゴール65まであと6）
12.3km・5'32"/km・心拍144bpm・68分。フィリピン帰国翌日。
体重は残り4.8kgを約21週で、週−0.23kgなら計画通り。
スコア74ptは回復ほぼ十分→イージー据え置きはドミノ通りの判断。
VO2max59→64kg到達時の理論値は約63。減量がそのまま有酸素エンジンの底上げになる。残り146日でゴール65なら「順調」。
→帰国翌日=感覚で「もう少し行けた」だったが、18kmは7/4まで封印ルールを守った。
```

## Step 7 — Substack 週次ログ行を生成する

形式:
```
| {YYYY-MM-DD} | {DISTANCE}km | {AVG_PACE}/km | {AVG_HR}bpm | {種別} | {体重}kg | 睡眠{X}h/{N}pt | VO2max{N} | {メモ1行} |
```

例:
```
| 2026-06-28 | 12.3km | 5'32"/km | 144bpm | イージー | 68.8kg | 睡眠6.5h/74pt | VO2max59 | 帰国翌日・違和感なし |
```

VO2max が無い回は `—` を入れる（列はずらさない）。

週次 Substack 記事の「今週の数字」セクションに貼り付ける用。

## エラーハンドリング

| 状況 | 対応 |
|---|---|
| 画像が添付されていない | 「Garminのスクリーンショットを貼り付けてください」と促す |
| 距離が読み取れない | その旨を明示して入力を求める |
| 日付計算が2026-06-12より前になる | 「Day番号が負またはゼロになっています。日付を確認してください」と出力 |
| 日付計算が164を超える | 「レース日（2026-11-22）を過ぎています。Day164が最終日です」と出力 |

## 注意事項

- **体重の累計マイナス値**は「スタート時100kgからの差分」で計算する（68kgなら-32kg）
- 走れなかった日の投稿（休養・ゼロデー）はこのスキルのスコープ外。ユーザーが文章を書く
- 過去日付の画像を処理する場合は、Step 1 で取得した今日の日付ではなく**画像内のタイムスタンプ**を使用する（Garmin 画像には撮影日が含まれることが多い）
- 勝田の教訓：本人が「もう少し行けた」と言っても、計画された距離・強度を超えた文章は書かない
